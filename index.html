Marketing QA Automation

Landing Page QA Automation

A simple browser automation workflow for checking landing pages, forms, screenshots and reports before performance marketing campaigns.

This is a generic public guide. Do not add private company URLs, real emails, API keys, passwords or internal information here.

What this solves

Manual checks

Opening pages, clicking CTAs, filling forms and taking screenshots can become repetitive.

Campaign risk

Sending paid traffic to a broken page or form can waste budget quickly.

Proof

The workflow saves screenshots and a CSV report so there is a clear trail of what happened.

How it works

1. Node.js runs the automation script.

2. The script calls agent-browser, which controls the testing browser.

3. A testing browser opens the landing page.

4. The workflow fills dummy data, clicks the approved CTA, captures screenshots and creates a report.

Tools used

Node.js
Runs the automation file.
PowerShell
Runs setup and test commands on Windows.
Notepad
Saves the prepared files.
agent-browser
Controls the testing browser.
Task Scheduler
Runs the test automatically.
AI / GPT
Helps create and troubleshoot the starter workflow.

Step 1 — Install Node.js

Download Node.js from:

Node.js link 
https://nodejs.org/en

Install the Windows LTS version with default options. Then open PowerShell and check:

PowerShell 
node -v
npm.cmd -v

If both commands show version numbers, Node.js is installed correctly.

Step 2 — Install browser control

This installs the browser-control component used by the automation.

PowerShell 
npm.cmd install -g agent-browser
agent-browser.cmd install

To test it:

PowerShell 
agent-browser.cmd --headed open
agent-browser.cmd close
Note: The testing browser may open as a blank page. That is normal.

Step 3 — Create the folder

PowerShell 
mkdir "$env:USERPROFILE\website-qa"
mkdir "$env:USERPROFILE\website-qa\config"

Final folder structure:

website-qa ├── run-page-test.js ├── run-test-now.bat ├── run-scheduled.bat ├── config │ └── page-config.json └── runs └── generated automatically after each run

Step 4 — Add page details

Open the config file:

PowerShell

Paste and update this example:

Step 5 — Create the main automation file

Open Notepad:

PowerShell 
notepad "$env:USERPROFILE\website-qa\run-page-test.js"

Paste the prepared JavaScript automation code into this file. Normal users should update the config file, not this main file.

Important: The full main automation code can be kept in a separate code section or separate file in this repository. Keep this public version generic and remove all private business details.

Step 6 — Create run files

Manual run file:

PowerShell 
notepad "$env:USERPROFILE\website-qa\run-test-now.bat"
run-test-now.bat 
@echo off
cd /d "%USERPROFILE%\website-qa"
node run-page-test.js
pause

Scheduled run file:

PowerShell 
notepad "$env:USERPROFILE\website-qa\run-scheduled.bat"
run-scheduled.bat 
@echo off
cd /d "%USERPROFILE%\website-qa"
echo ======================================== >> "%USERPROFILE%\website-qa\scheduler-log.txt"
echo Website QA started at %date% %time% >> "%USERPROFILE%\website-qa\scheduler-log.txt"
node run-page-test.js >> "%USERPROFILE%\website-qa\scheduler-log.txt" 2>&1
echo Website QA finished at %date% %time% >> "%USERPROFILE%\website-qa\scheduler-log.txt"
echo ======================================== >> "%USERPROFILE%\website-qa\scheduler-log.txt"
Important: In Notepad, choose Save as type: All Files. Otherwise Windows may save the file as .bat.txt, which will not run correctly.

Step 7 — Run the test

PowerShell 
cd "$env:USERPROFILE\website-qa"
node ".\run-page-test.js"

Or double-click:

Run file 
C:\Users\<YourName>\website-qa\run-test-now.bat

Step 8 — Check output

Every run creates a dated folder here:

Folder 
C:\Users\<YourName>\website-qa\runs

The run folder can include:

  • Before screenshot
  • After screenshot
  • CSV / Excel report
  • Time taken
  • Pass / fail status
  • Manual email-check note

Step 9 — Schedule it

Open Windows Search and type:

Windows Search 
Task Scheduler
  • Create Basic Task
  • Choose weekly or twice weekly
  • Choose a time during working hours
  • For Action, choose Start a program
  • Select run-scheduled.bat
Scheduling note: The PC must be switched on, connected to the internet and available at the scheduled time. If the PC is off, asleep or not logged in, the scheduled test may not run properly.

Important reminder

For every new landing page, do not rewrite the main automation file. Update only:

Config file 
C:\Users\<YourName>\website-qa\config\page-config.json
