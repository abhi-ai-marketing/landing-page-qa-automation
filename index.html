Workflow

A simple browser automation workflow for checking landing pages before paid traffic goes live. It helps test the key page journey, capture screenshots, record results and keep checks repeatable.

Browser automation Landing page checks Screenshots + report Scheduled testing

Pre-launch checks

  • Page loads correctly
  • Primary CTA is visible
  • Form fields are detected
  • Dummy test details are submitted
  • Screenshots and report are saved
What this solves

Landing page checks are important, but often repetitive.

Performance campaigns depend on working landing pages, CTAs and forms. This workflow turns basic pre-launch checks into a repeatable process.

Challenge 01

Manual checks

Repeatedly opening pages, filling forms and taking screenshots can take time.

Challenge 02

Campaign risk

Paid traffic should not be sent to a broken page, CTA or form journey.

Challenge 03

Review trail

Saved screenshots and reports make it easier to review what happened in each run.

A useful safety check before campaign traffic starts.

Workflow

How the automation runs

The workflow opens a selected landing page, checks the required page elements, submits dummy test details and saves the output.

1

Set up the tools

Install Node.js and the browser-control component using PowerShell.

2

Create the working folder

Create one local folder to store scripts, config files, screenshots and reports.

3

Add page details

Update the config file with page name, URL, test details and button labels.

4

Run the test

The testing browser opens the page, fills the form and clicks the approved CTA.

5

Save evidence

The workflow saves screenshots, page snapshots and a simple CSV report.

6

Repeat on schedule

The same test can be scheduled to run weekly, twice weekly or before campaign launch.

Simple logic behind the workflow

PowerShell starts the script. Node.js runs the script. The script calls agent-browser. agent-browser controls the testing browser. The workflow reads the config, opens the page, performs the checks, then saves screenshots and a report.

Setup snippets

Commands and config examples

The page details can be changed in the config file, so the same workflow can be reused for different landing pages.

Check Node.js

PowerShell 
node -v
npm.cmd -v

Install browser control

PowerShell 
npm.cmd install -g agent-browser
agent-browser.cmd install

Create folder

PowerShell 
mkdir "$env:USERPROFILE\website-qa"
mkdir "$env:USERPROFILE\website-qa\config"

Run test

PowerShell 
cd "$env:USERPROFILE\website-qa"
node ".\run-page-test.js"

Example config

page-config.json 
{
  "pageName": "Example Landing Page",
  "pageUrl": "https://www.example.com/landing-page/",
  "testName": "Website QA Test",
  "testEmail": "test@example.com",
  "testPhone": "07000000000",
  "sourceValue": "Google",
  "submitButtonLabels": ["Submit", "Download", "Send"],
  "successText": ["thank", "success", "submitted"]
}

Manual run file

run-test-now.bat 
@echo off
cd /d "%USERPROFILE%\website-qa"
node run-page-test.js
pause

Automation

Run the check on a schedule

Once the manual run works, Windows Task Scheduler can run the workflow weekly, twice weekly or before campaigns go live.

1. Open Task Scheduler

Use Windows search and open Task Scheduler.

2. Create task

Create a basic task and choose the schedule.

3. Select run file

Select the .bat file that starts the automation.

Scheduling requirement

The PC must be switched on, connected to the internet and available when the scheduled test runs. If the PC is off or asleep, the scheduled test may not run.
