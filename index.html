What this solves

Manual landing page checks are important, but repetitive.

Performance marketers often test the same journey again and again: open page, click button, fill dummy data, submit form, take screenshots and check the result.

Problem 01

Manual checks

Repeating the same checks before campaigns takes time.

Problem 02

Campaign risk

Sending paid traffic to a broken page or form can waste budget quickly.

Problem 03

No clear trail

Screenshots and a report make the test easier to review later.

Why now

Browser automation is not new. Access is the change.

Tools like Selenium and Playwright have existed for years, but they mostly sat with developers and QA teams. AI makes it easier for marketers to understand the workflow, create starter code and troubleshoot the setup.

Old way

Workflow

How the workflow runs

The test itself is normal browser automation. AI helps create and refine the starter workflow.

1

Install the basics

Install Node.js, then install the browser-control component through PowerShell.

2

Create the working folder

Create one local folder called website-qa to store files, outputs and reports.

3

Add the prepared files

Save the main automation file, config file and run file using Notepad.

4

Update page details

Change the page name, page URL, test email and approved button wording in the config file.

5

Run the test

The browser opens, the page loads, dummy data is filled, the approved CTA is clicked and screenshots are saved.

6

Review the output

Each run creates a dated folder with screenshots, pass/fail status, time taken and report.

Simple logic behind the code

PowerShell starts the script. Node.js runs the script. The script calls agent-browser. agent-browser controls the testing browser. The workflow reads the config, opens the page, performs the checks, then saves screenshots and a report.

Copy snippets

Setup commands and files

These are the small setup snippets used to create the local workflow.

Check Node.js

PowerShell 
node -v
npm.cmd -v

Install browser control

PowerShell 
npm.cmd install -g agent-browser
agent-browser.cmd install

Create folder

PowerShell 
mkdir "$env:USERPROFILE\website-qa"
mkdir "$env:USERPROFILE\website-qa\config"

Run test

PowerShell 
cd "$env:USERPROFILE\website-qa"
node ".\run-page-test.js"

Example config

page-config.json 
{
  "pageName": "Example Landing Page",
  "pageUrl": "https://www.example.com/landing-page/",
  "testName": "Website QA Test",
  "testEmail": "test@example.com",
  "testPhone": "07000000000",
  "sourceValue": "Google",
  "submitButtonLabels": ["Submit", "Download", "Send"],
  "successText": ["thank", "success", "submitted"]
}

Manual run file

run-test-now.bat 
@echo off
cd /d "%USERPROFILE%\website-qa"
node run-page-test.js
pause

Automation

Make it automatic with Task Scheduler

Once the manual run works, the same test can be scheduled weekly, twice weekly or before campaigns go live.

1. Open Task Scheduler

Use Windows search and open Task Scheduler.

2. Create task

Create a basic task and choose the schedule.

3. Select run file

Select the .bat file that starts the automation.

Important scheduling note

The PC must be switched on, connected to the internet and available when the scheduled test runs. If the PC is off or asleep, the scheduled test may not run properly.
