The Landing Page QA Automation Guide

A simple browser automation workflow to test landing pages before performance campaigns go live. It helps check the page, the CTA, the form flow, the screenshots and the report — all in one repeatable process.

What this helps with

Before sending paid traffic to a page, it is useful to have a quick, repeatable check so obvious issues do not slip through.

Quick overview

  • Open a landing page automatically
  • Check the key CTA and form flow
  • Fill dummy test details
  • Capture before and after screenshots
  • Measure time taken
  • Generate a simple report
  • Run it again on a schedule
What this solves

A simple safety check before performance traffic goes live

Manual checks

Opening pages, clicking CTAs, filling forms and taking screenshots can become repetitive very quickly.

Campaign risk

Sending paid traffic to a broken page, broken button or broken form can waste budget fast.

Clear output

The workflow saves screenshots and a simple report so there is a visible record of what happened.

Workflow

How the workflow works

1

Install the basics

Install Node.js, then install the browser-control component. This sets up the environment required to run the test locally.

2

Create the files

Save the prepared automation file, the config file and the manual run file in one local working folder.

3

Update page details

Add the page name, page URL, dummy name, dummy email and dummy phone in the config. For a new page, only this part changes.

4

Run the test

The script opens the page, checks the CTA and form, enters dummy details, submits, captures screenshots, measures time and creates the report.

5

Review the output

Each run creates a timestamped folder with screenshots, pass/fail results and the overall time taken.

6

Schedule it

Once the manual test works, Windows Task Scheduler can run the workflow weekly or twice weekly.

Simple logic behind the code

PowerShell starts the script. Node.js runs it. agent-browser controls the testing browser. The script reads the page details from the config file, opens the page, performs the test steps, then saves the outputs into a run folder.

Files

Files used in the setup

run-page-test.js

The main automation file. This contains the workflow logic that runs the landing page test.

page-config.json

The config file. This stores the page URL, page name and dummy test details.

run-test-now.bat

A shortcut file so the test can be run without typing the full command every time.

Example run command

cd "%USERPROFILE%\website-qa"
node run-page-test.js

Example manual run file

@echo off
cd /d "%USERPROFILE%\website-qa"
node run-page-test.js
pause
Output

What the output looks like

Saved screenshots

The workflow captures screenshots before and after the form submission so the result can be checked visually.

Simple report

A report is generated showing key checks, overall status, time taken and where the screenshots were saved.

Timestamped folders

Each test run creates a separate dated folder, so old outputs are not overwritten.

Manual email check

If the workflow triggers an email, that part may still need to be checked manually in the inbox.

Automation

How to schedule it

Use Windows Task Scheduler

After the manual run works properly, the same workflow can be automated so it runs weekly or twice weekly without manual effort.

Open Windows search and type: Task Scheduler
Click: Create Basic Task
Choose: Weekly or Twice weekly
Select the manual run file or scheduled run file
Set it during working hours
Ensure the PC is switched on, connected to the internet and available when the task is due to run
